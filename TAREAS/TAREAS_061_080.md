### TAREA 061 · 🟠 ALTO
**Importación masiva de catálogo desde CSV o Excel**

**Descripción:** Permitir cargar o actualizar muchos productos de una sola vez subiendo un archivo CSV, en vez de cargarlos uno por uno.

**Prompt para el agente:**
```
Leé el archivo CLAUDE.md para contexto del proyecto.

1. Verificá primero: no existe hoy ningún endpoint ni UI de importación masiva de productos.
   Ya existe `src/app/api/products/bulk/route.ts` (PATCH) para edición masiva de productos
   YA EXISTENTES por ID — esta tarea es distinta: es alta/actualización de productos a partir de
   un archivo, identificando cada fila por `productCode`.
2. Creá un endpoint `POST /api/products/import` que reciba un array de filas ya parseadas en el
   body (el parseo del CSV se hace en el cliente, no envíes el archivo binario al backend):
   cada fila con `productCode, name, categoryName, costPrice, salePrice, ivaRate, unit, stock,
   minStock` (usá los mismos nombres de campo que `CreateProductInput`/`UpdateProductInput` en
   `src/modules/products/product-validation.ts`).
   - Si `productCode` coincide con un producto existente del tenant, actualizalo (reusando
     `validateUpdateProductInput`).
   - Si no coincide con ninguno, creá un producto nuevo (reusando `validateCreateProductInput`).
   - Procesá todo dentro de una transacción de Prisma, fila por fila, acumulando errores por fila
     sin abortar el resto (cada fila inválida se reporta, las válidas se aplican igual).
   - Devolvé un resumen: `{ created: number, updated: number, errors: Array<{ row: number,
     message: string }> }`.
3. En `src/app/ui/products-inventory.tsx`, agregá un botón "Importar CSV" que abra un modal con:
   - Un input de archivo (`accept=".csv"`).
   - Parseo simple del CSV en el cliente (split por líneas y comas, sin librerías nuevas — ya se
     usa este patrón manual en el proyecto para exportar CSV, aplicá el inverso para importar; la
     primera fila del archivo son los encabezados de columna).
   - Una vista previa de las primeras filas antes de confirmar.
   - Al confirmar, `POST /api/products/import` con las filas parseadas, y mostrá el resumen
     resultante (creados, actualizados, errores por fila) al usuario.
4. Agregá también un botón "Descargar plantilla CSV" que descargue un archivo de ejemplo con los
   encabezados correctos y una fila de muestra (mismo patrón de `Blob`/`URL.createObjectURL` ya
   usado en el proyecto para exportar CSV).

RESTRICCIONES ESTRICTAS:
- No modifiques el schema de Prisma ni `PATCH /api/products/bulk` (edición masiva existente).
- No agregues librerías externas de parseo de CSV/Excel; parseo manual de CSV es suficiente
  (no hace falta soportar .xlsx binario en esta tarea).
- Reusá `validateCreateProductInput`/`validateUpdateProductInput` — no dupliques reglas de
  validación de producto.
- Ejecutá `npm run build` al final.
```

**Archivos afectados:** `src/app/api/products/import/route.ts` (nuevo), `src/app/ui/products-inventory.tsx`
