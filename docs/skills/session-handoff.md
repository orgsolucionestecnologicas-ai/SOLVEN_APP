# session-handoff
Apply this skill at the end of every significant work session. Purpose: preserve full context so the next session starts without confusion.

## Required Sections

### Completed
List every task finished this session. Be specific.
Example: Added customer name join to GET /api/debts response

### Files Changed
List every file modified, created, or deleted.
Example: src/app/api/debts/route.ts

### Commits Made
List commit hashes and messages in order.
Example: a1b2c3d feat: surface customer name in debts API

### Decisions Made
Document any non-obvious choices and why they were made.

### Risks or Blockers
List anything incomplete, risky, or unclear.

### Validation Status
Report result of: lint / typecheck / prisma validate / tests / git status

### Next Task
State the exact next recommended task. One task only. Be specific.

## Rules
- Handoff must be delivered before closing the session
- Do not summarize vaguely — be exact
- Working tree must be clean before handoff
- Do not recommend multiple next tasks — one only
